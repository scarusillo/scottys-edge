## Step 1 — Read the auto-generated prompt (source of truth)

Read `data/cards/kling_prompt.txt` first.

**Freshness check:** the file mtime MUST be from today (the same calendar date as
the latest `graded_at` in the DB). If the file is older than today's latest grade,
it is stale — STOP and tell the user to run `python main.py grade --email` (or
regenerate via `_generate_kling_prompt`). Do not fall back to hand-authoring.

## Step 2 — Verify the record against the DB

Run this command and compare its output to the record in `kling_prompt.txt`:

```bash
PYTHONIOENCODING=utf-8 python -c "
import sqlite3
con = sqlite3.connect('data/betting_model.db')
r = con.execute('''SELECT SUM(CASE WHEN result=\"WIN\" THEN 1 ELSE 0 END),
       SUM(CASE WHEN result=\"LOSS\" THEN 1 ELSE 0 END), SUM(pnl_units)
FROM graded_bets WHERE DATE(created_at) >= \"2026-03-04\"
AND result NOT IN (\"DUPLICATE\",\"PENDING\",\"TAINTED\") AND units >= 3.5''').fetchone()
w, l, pnl = r
print(f'Season: {w}W-{l}L +{pnl:.1f}u {round(w/(w+l)*100,1)}%')"
```

The season record (W-L and %) MUST match the numbers in `kling_prompt.txt`.
If they don't match, STOP and tell the user. Do not invent a number to split the difference.

## Format rules (when editing for style)

- Duration: 12 seconds (Sora, NOT Kling — filename is legacy)
- 9:16 vertical
- All text on-screen, NEVER voiceover, no spoken audio
- Athletes fully visible and well-lit — NOT silhouettes
- Athletes must be IN MOTION performing their sport (swinging, skating, shooting, pitching)
  — not static poses
- Sport spotlight colors: baseball = red, hockey = blue, soccer = golden, basketball = orange
- LED scoreboards display all numbers

## What NOT to do

- Do NOT write record numbers from memory
- Do NOT round/re-round the win %
- Do NOT use a "projected" or "forward-looking" record
- Do NOT change "WIN"/"LOSS" labels to "W"/"L" when querying DB
